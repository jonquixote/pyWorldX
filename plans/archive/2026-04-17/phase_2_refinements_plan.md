## Task 1: Analytical Stability & Human Capital Calibration (Q79, Q80)

**Status:** Done
**Effort:** 1-2 days

### Problem
The `HumanCapitalSector` runs at the default $dt=1.0$ (`timestep_hint = None`), causing integration inaccuracies that force analytical tests to use loose tolerances (0.1 relative). Furthermore, the pre-industrial equilibrium calibration where $dH/dt = 0$ at $H=0.3$ is not mathematically perfect.

### Implementation Steps
1. **Substep Ratio:** In `pyworldx/sectors/human_capital.py`, change `timestep_hint = None` to `timestep_hint = 0.015625` (1/64 substeps) or `1/512` if required for stability.
2. **Equilibrium Calibration:** Ensure that at pre-industrial `service_output_per_capita`, `EducationRate` exactly equals `SkillDegradationRate` at $H=0.3$. Adjust the `_EDU_RATE_Y` lookup table slightly so the baseline evaluates to $0.3 \times \frac{\ln(2)}{10.0} \approx 0.02079$.
3. **Tighten Tests:** In `tests/unit/test_human_capital.py`, restore the `1e-4` hybrid tolerance for the analytical decay test to ensure the RK4 integrator correctly tracks the high-frequency ODE.

## Task 2: Production Function Refactoring (Q81, Q71)

**Status:** Not Started
**Effort:** 2-3 days

### Problem
The Capital and Agriculture sectors use simplified heuristics or end-of-pipe multipliers instead of true Cobb-Douglas production functions. Q81 mandates $H$ must be part of the Cobb-Douglas equation with 50-60% elasticity. Q71/Q84 mandates Agriculture uses an Energy (60%), Materials (20%), Phosphorus (20%) split.

### Implementation Steps
1. **Capital Sector:** Modify `pyworldx/sectors/capital.py`.
   - Replace `io = ic * (1.0 - fcaor) * cuf / self.icor * h_mult` with a Cobb-Douglas formulation: $Q = A \cdot K^\alpha \cdot R^\beta \cdot H^{(1-\alpha-\beta)}$.
   - Set $\alpha \approx 0.25$, $\beta \approx 0.20$, and $1-\alpha-\beta \approx 0.55$.
   - Re-calibrate $A$ (Total Factor Productivity) to maintain the ~1900 baseline Industrial Output of $6.65 \times 10^{10}$.
2. **Agriculture Sector:** Modify `pyworldx/sectors/agriculture.py`.
   - Update the Land Yield Multiplier (`LYMC`) computation to use the Cobb-Douglas form: $Yield = A \cdot E^{0.60} \cdot M^{0.20} \cdot P^{0.20}$, where $E$ is Energy input, $M$ is Material (Capital) input, and $P$ is Phosphorus availability.
3. **Update Tests:** Fix integration tests in `test_world3.py` and `test_phase1_integration.py` to account for the new production curves.

## Task 3: EIA Energy Baseline Calibration (Q84, Q77)

**Status:** Not Started
**Effort:** 2-3 days

### Problem
Energy demands broadcasted to the `CentralRegistrar` are abstract dimensionless variables. They must be calibrated to real-world EIA physical units, such that global energy consumption matches the modern ~600 EJ/yr baseline.

### Implementation Steps
1. **Energy Intensity Parameters:** Add `energy_intensity` parameters to all demand-generating sectors (Capital, Service, Agriculture, Technology Energies, etc.).
2. **Baseline Scaling:** Calibrate these intensities so that `total_demand` in the `CentralRegistrar` equates to ~600 EJ/yr at the 2020 timestep.
3. **Ceiling Verification:** Ensure `total_supply` generated by the Energy sectors scales appropriately in EJ/yr, maintaining the 65% thermodynamic ceiling mathematically.

## Task 4: USDA SSURGO SOC Limits & Phosphorus (Q84, Q71)

**Status:** Not Started
**Effort:** 3-4 days

### Problem
The model uses a rough `14000` Mt global phosphorus estimate and lacks the "living matrix" of Soil Organic Carbon (SOC). We need the "rooting depth resilience threshold" that triggers agricultural collapse when soil degrades.

### Implementation Steps
1. **SOC Stock:** Add a new state variable `SOC` (Soil Organic Carbon) to `pyworldx/sectors/phosphorus.py` (or agriculture).
2. **SOC Degradation:** Make $d(SOC)/dt$ negative when pollution is high or when intensive industrial farming (high FIOAA) depletes the matrix.
3. **Rooting Depth Threshold:** In `agriculture.py`, introduce a threshold (e.g., $SOC < SOC_{critical}$). Below this threshold, the Land Yield Multiplier (`LYMC`) non-linearly drops toward zero, regardless of how much chemical phosphorus ($P_{soc}$) is available.
4. **Parameter Initialization:** Document the USDA SSURGO reference for the starting parameters.

## Task 5: 5-Stock Global Carbon Model (Q84, Q75)

**Status:** Not Started
**Effort:** 4-5 days

### Problem
`PollutionGHGModule` uses a single `ghg_stock` with a 100-year delay. This cannot reproduce real NOAA/GCB flux partitioning or Gaian feedback loops (e.g., permafrost thaw).

### Implementation Steps
1. **Model Rewrite:** In `pyworldx/sectors/pollution_ghg.py`, replace `ghg_stock` with 5 distinct state variables:
   - `C_atm` (Atmosphere)
   - `C_land` (Land Biomass)
   - `C_soc` (Soil Organic Carbon — link to Task 4)
   - `C_ocean_surf` (Ocean Dissolved Carbon)
   - `C_ocean_deep` (Deep Ocean/Sediment)
2. **Flux Equations:** Define the exchange rates ($k_{atm \to ocean}$, $k_{atm \to land}$, etc.) based on Global Carbon Budget 2024 partitions.
3. **Gaian Feedback:** Make the land/ocean sink rates temperature-dependent (reading $T$ from `ClimateSector`). If $T$ rises significantly, `C_soc` outgasses back into `C_atm`.
4. **Climate Driver:** `ClimateSector` must be updated to read `C_atm` instead of the old `ghg_stock` for its radiative forcing computation.
5. **Tests:** Update all GHG tests to verify mass conservation across the 5 compartments.
